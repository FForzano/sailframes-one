E1 Fleet Tracker — Bill of Materials

Complete parts list to build one E-series SailFrames fleet tracker — the same hardware running on six J/80s and Sonars in Boston Harbor. Two GNSS options, one common chassis, ~$165–280 per unit. Apache 2.0.

Prices approximate as of May 2026 and will drift. Source links are Amazon search URLs (specific listings rot — you'll want to verify before ordering).

Option A — Quectel LG290P (PPK, ~2 cm post-processed)

#PartQtyApprox. $Source
A1Waveshare LG290P GNSS Module with active antenna1~$109Amazon · Waveshare
A2(Optional) external high-gain GNSS antenna with SMA, ~28 dB LNA1$15–25Amazon

Notes

  • The stock antenna in the Waveshare kit works fine for fleet use; the external one is an upgrade for boats with metal arches blocking sky.
  • Quectel firmware AANR01A06S is required for the two-parameter PQTMCFGMSGRATE syntax used by the E1 boot config.
  • First-time configuration is easiest with QGNSS on Windows; PyGPSClient on macOS has limited LG290P support.
  • PPK workflow: record .rtcm3 → download CORS base obs from geodesy.noaa.gov/UFCORS (free) → RTKLIB RTKPOST → fixed solution.

Option A GNSS subtotal: ~$110.

Option B — Quectel LC29H (dual-band, no PPK, sub-meter)

The closest cousin to the LG290P in the same Quectel family. Same UART interface, but no raw observable output for PPK — you use the NMEA fix directly.

#PartQtyApprox. $Source
B1Waveshare LC29H (DA) GNSS Module with active antenna1~$30–40Amazon · Waveshare
B2(Optional) external L1/L5 active antenna with SMA1$15–25Amazon
⚠ PCB compatibility caveat. PCB v1.1's J2 GNSS connector was laid out for the LG290P pinout specifically. The LC29H breakout has a similar UART idea but the connector pitch and pin order differ — for an Option B build you will need either a small adapter cable between J2 and the LC29H header (4 wires: VCC / GND / TX / RX), or a respin of the PCB with an LC29H-compatible J2 footprint.

Variants to be aware of

  • LC29H (DA) — dual-band L1+L5, no RTK, multi-constellation. Use this.
  • LC29H (BS) — has RTK. Works too, but you don't need the extra cost.
  • LC29H (CA) — single-band L1 only. Skip.

Realistic accuracy expectations

  • Manufacturer typical: CEP ~1.5 m standalone, CEP ~50 cm with SBAS in good sky-view conditions.
  • Dual-band L1+L5 still helps measurably: better multipath rejection near rigging and shore reflections than a single-band module.

Option B GNSS subtotal: ~$30–40.

Display (O1)

The on-water TFT showing speed, COG, battery, and GPS status. Without it the device runs headless — recording still works, but you only see status over USB serial or via the Wi-Fi auto-upload counts. The shipping firmware initializes the TFT at boot and assumes its presence; a headless build needs the TFT init code disabled or wrapped in a #define HAS_TFT guard.

DPS310 (O2)

Weather-trend logging at 0.1 Hz (do not expect gust detection). If installed inside the IP67 enclosure, also fit the VENT-PS1 vent — a sealed box reads internal pressure, not ambient.

Sourcing notes

  • Quectel modules (LG290P / LC29H) — Amazon stock fluctuates; Waveshare's own store and Robu.in are reliable backups. The important part is "Waveshare-branded module" — bare Quectel modules without breakout don't have headers or antenna.
  • ESP32 board — must be the ELEGOO V1 with the CP2102 USB chip. CH340-variant boards work electrically but the firmware flashing scripts (scripts/flash-edge.sh) assume CP2102 enumeration.
  • TFT — must be the ST7796U controller, 480 × 320, SPI. ILI9486 variants of the same physical size will need the User_Setup.h for TFT_eSPI swapped, and ILI9341 (320 × 240) is the wrong resolution.
  • microSD module — ensure it has 3.3 V level shifters on the data lines, not bare 5 V passthrough. Cheap modules without level shifters work intermittently and corrupt cards.
  • LiPo cell — 906090 is the cell footprint (9 mm × 60 mm × 90 mm). Other 6000 mAh cells fit if their dimensions match the YETLEBOX internal volume.
  • Enclosure — any IP67 ABS clear-lid box ~115 × 90 × 55 mm with M2.5 internal mounting bosses works; YETLEBOX is the one the fleet uses.

Firmware & software

Same firmware (edge-e/firmware/sailframes_edge/sailframes_edge.ino) runs on both options. At boot the firmware sends LG290P-specific PQTM* config; on an LC29H board those commands return ERROR,1 and are silently ignored — the LC29H starts in its default NMEA mode, which is exactly what Option B wants. No code change required for Option B beyond skipping PPK download / RTKLIB in post-processing.

Source — firmware, KiCad files, build scripts — github.com/sailframes/core/edge-e. Apache 2.0.
